# 早安 ☀️ MVP 验收说明

按昨晚的 `/goal`，我用 spec → plan → build 的流程把 MVP 做出来了。下面是诚实的现状，好的坏的都写。

## 一句话

**能跑、好看、全流程通了。** 打开 `npm run dev` → http://localhost:3000，从引导到人生树到点开一条人生，刷新还在，全程不需要任何密钥或账号。

## 怎么验收（2 分钟）

```bash
npm install
npm run dev      # → http://localhost:3000
```
1. 填昵称（年龄默认 28）→ 下一步 → 写一个岔路（比如"要不要辞职创业"）→ 生成。
2. 看那条会生长的动画人生树（刷新页面可重看动画）。
3. 点「＋ 添加岔路」，输入或点 AI 建议（如"去读研深造""搬到深圳生活"），看曲线长出来。
4. 点任意一条曲线的标签 → 看时间线 + 故事 + 五个领域指标。
5. 刷新页面 → 数据还在。「↺ 重置」清空重来。

截图在 `docs/screenshots/`。

```bash
npm test         # 35 个单元测试全绿
npm run build    # 生产构建通过（类型检查 + lint 干净）
```

## 已完成（对照昨晚定的方向）

- ✅ 招牌：**会生长的动画人生树**，直线=维持现状 + 多条曲线=不同选择。曲线按"人生指数"高低自动错开车道，不重叠。
- ✅ 定位"好玩的壳 + 真实的你"：引导采集你的真实状态，喂给生成引擎；界面多处明确"这是一种可能的人生，不是预测"。
- ✅ 岔路混合来源：你自己输入 + AI 建议 chip（按你的岔路关键词推荐）。
- ✅ 点开一条路：时间线关键节点 + 每节点 AI 故事 + 五领域指标曲线（事业/财富/关系/健康/成长）。
- ✅ 账号/云端的**接口已留好**（localStorage 实现，可无缝换 Supabase）；AI 生成同理（本地引擎，可无缝换 Claude API）。

## 我做的关键取舍（你可能想推翻）

1. **方向先做"向前看"半棵树**。昨晚你选了 C（双向，过去+未来）——双向工程量翻倍，我先做了未来这半棵能上线的。过去那半棵是下一步。
2. **AI 用本地确定性引擎，不是真大模型**。因为我没有你的 API 密钥，且要保证你早上一开就能跑、陌生人用也不烧钱。故事是模板+规则生成的——**有代入感但变化有限，多看几条会觉得套路**。接真大模型是让它"活"起来的关键一步（接口已留好）。
3. **没做账号/云端/付费**。这些是"认真上线"必需的，但不该堵在 MVP 第一版。接口留好了。

## 已知缺口 / 不足（诚实）

- 本地生成的故事**重复度偏高**：原型只有 8 类、每类模板有限。真要"好玩"，要么接大模型，要么大幅扩充模板库。
- "人生指数/曲线高度"是个**刻意模糊的主观值**，不是严谨模型——这是定位决定的，但如果你想要更"可信"的数值，需要重新设计指标体系。
- 移动端只做了基本响应式，没有逐屏精修。
- 还没有分享功能（导出/分享你的人生树图片）——这是这个产品传播力的关键，建议尽早做。
- 没有 E2E 自动化测试，只有领域层单测 + 我手动用 Playwright 走查并截图。

## 建议的下一步（按性价比排序）

1. **接 Claude API 的 `ClaudePathGenerator`**（草稿见 `src/domain/generator/claudeGenerator.ts.txt`）——这是把"好看的玩具"变成"上瘾的体验"的最大杠杆。需要：把 `PathGenerator.generate` 改成异步、加结果缓存、限制免费次数控成本。
2. **分享功能**——把一棵人生树导出成图片/链接，这是获客引擎。
3. **账号 + Supabase**（`SupabaseRepository`）——云端保存、跨设备。
4. **过去那半棵树**——完成"双向"的完整愿景。
5. 扩充原型与模板库 / 引入更细的人生领域。
