## 需要你来的部分(我今晚做不了,代码已备好)

### 1) 开启云同步(最重要 —— 解决"清缓存就丢数据")
按 `docs/supabase-setup.md`:
1. 建一个 Supabase 项目;SQL Editor 跑文档里的建表 + RLS SQL。
2. Authentication 开 Email 登录;URL 配置里把 Site/Redirect URL 设成你的地址(如 `http://localhost:3000`)。
3. 在 `.env.local` 加两行(从 Project Settings → API 拿):
   ```
   NEXT_PUBLIC_SUPABASE_URL=...
   NEXT_PUBLIC_SUPABASE_ANON_KEY=...
   ```
4. 重启 → 侧边栏底部出现「云同步」→ 邮箱魔法链接登录 → 本地数据会自动迁到云端,多端同步。
> 诚实提醒:云端路径**没在真 Supabase 上联调过**(我没有你的项目),请第一时间验一遍登录 + 跨设备。flag 关着时对现有 app 零影响。

### 2) 真·后台推送(关着页面也能提醒)
现在的提醒只在**页面开着**时响。要做到关掉也提醒,需要 **VAPID 密钥 + 后端推送**(存订阅 + 服务端发推),SW 里再加 `push`/`notificationclick` 处理。要的话给我开绿灯,我做。

### 3) Google/Outlook 日历**双向**同步
今晚的 ICS 是**只读导入**(无需 OAuth)。双向(在你真实日历上写/改)需要 **OAuth client id/secret + 后端 token 管理**,要你的 Google Cloud 凭据。
