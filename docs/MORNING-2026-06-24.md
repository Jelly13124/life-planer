## 怎么在手机/模拟器上跑（重要：连接方式）
1. 电脑上：`cd mobile && npx expo start`
2. 模拟器连不上时用这招（昨晚踩过坑）：`adb reverse tcp:8081 tcp:8081`，然后开 `exp://127.0.0.1:8081`。真机用 Expo Go 扫码即可。
3. 想让 **AI 增强分支 + 和未来的自己对话** 生效：电脑开着网页 dev server，并在 `mobile` 设环境变量 `EXPO_PUBLIC_API_BASE_URL=http://<你电脑局域网IP>:3000`。没设就离线降级（分支用本地预测、对话提示"需要连接后端"），不报错。

## 诚实说明：两件需要 dev build（非 Expo Go）的事 —— 建议和 TestFlight 一起做
1. **本地通知**：`expo-notifications` 在 Expo Go(SDK 53+) 一加载就崩（远程推送被移除）。我已加 Expo Go 检测让它在 Expo Go 里**整体 no-op 不崩**；真正的到点提醒只在 **dev build / TestFlight** 里生效。
2. **拖拽排期（P7，唯一没做的）**：你选的是拖拽式。但它需要 gesture-handler + reanimated 的 worklet/babel 正确配置，而当前 reanimated 是传递依赖、没配 babel 插件——在 Expo Go 里一旦触发拖拽手势很可能崩，且 adb 没法可靠模拟"长按再拖"来提前验证。所以我**没有冒险塞进去**，保住整个 app 的稳定。**轻点选时间排期是完全可用的替代**（点未排任务→选时间→落到时间轴）。拖拽放到 dev build 阶段（那时 reanimated/通知/TestFlight 一起配好）再加。

→ 结论：Expo Go 里今天能完整体验「安排时间轴+轻点排期+月年日历+目标进度+人生树加分支+对话屏」；通知和拖拽等做 dev build 时点亮。
