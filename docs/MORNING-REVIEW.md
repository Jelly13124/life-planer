# 早安 ☀️ 完整 PRD 验收说明（v2）

按昨晚的 `/goal`，把 `docs/PRD-prediction-experience.md` 的功能尽量做全了，是一个**完整可用的 app**。下面是诚实的现状——好的坏的、验证到什么程度，都写清楚。

## 一句话
**全流程跑通、能用、好看。** 引导 → 会生长的**递归人生地图** → 点开一条路看**多维度时间线**、切**乐观/最可能/保守** → **和未来的你对话** → 节点上**再长岔路** → 右下角**规划助手**。默认 `deepseek-chat`，没密钥也能用（本地兜底）。

## 怎么验收（5 分钟）
```bash
npm install
npm run dev      # → http://localhost:3000   （.env.local 里已配 DEEPSEEK_API_KEY）
```
1. 填真实的你（昵称/年龄/学历/职业/**所在地**/**身份(如 H1B)**/情感/纠结的岔路）。
2. 看**人生地图**：从"现在"长出曲线（拖动平移、滚轮缩放）。等几秒，AI 会把每个节点写成扎根现实的真事。
3. **点曲线**进详情：时间线每个节点带**维度标签**（事业/财务/身份/感情…）；顶部切**乘观/最可能/保守**三种走向。
4. 点「✨ 和 X 岁的你聊聊」：跟走了这条路的未来自己第一人称对话；聊出新选择能「＋加进人生树」。
5. 回地图，**点某条路上的节点** → 在那一年**再分一条岔路**（递归树）。
6. 右下角「💬 规划助手」：帮你理清纠结、提没想到的路，也能一键加进树。

截图在 `docs/screenshots/v2-*.png`。

## 对照 PRD，做了什么
- **P1 真实+细致+多维**：每条路 6–10 个节点、每段含真实锚点（实测一条"跳槽大厂"自动写出了 *入职Meta适应期 → I-140批准 → 绿卡获批购房 → 换组与感情波折 → 职业瓶颈 → 副业起步*，覆盖事业/财务/身份/感情/健康，严格从现在往后、不与"读完研/H1B"矛盾）。**这就是你之前骂"假/不细致"的根治。**
- **P2 多走向**：详情页切 乐观/最可能/保守，按需生成、不重复。
- **P3 / R5a 和未来的你对话**：`/api/chat`，第一人称、扎根那条人生。
- **R6 递归人生地图**：时间走 X 轴的可平移缩放地图；点节点能在那一年再长岔路；非活跃分支淡化。
- **R7 对话解锁岔路**：聊天里把新选择一键加进树（作为那条路的子分支）。
- **P4 规划助手**：常驻浮窗（`/api/assistant`）。
- **§12 Odyssey 三路**：加岔路时分"替代路 / 疯狂路"。
- 其它：localStorage 升到 v2（旧的不兼容数据自动丢弃）；50 个单测全过；生产构建干净。

## 验证到什么程度（诚实）
- **我亲手在浏览器里跑通的**：引导全流程、递归地图渲染 + 真实 AI 节点、Odyssey 加岔路面板、规划助手**真实 DeepSeek 往返**（截图为证）。
- **构建+类型检查+代码审查过、但没用自动化点穿的**：点开路径详情、走向切换、"和未来的你"对话、点节点生成子分支。原因：Playwright 很难可靠点击 SVG 里的曲线/节点（有平移手势 + 标签 pointer-events），但**真人用鼠标点是正常的**。这几处逻辑都已类型检查通过、我也读过代码。早上你点一下就能确认。

## 已知不足 / 边界（诚实）
- 仍是"**一种可信的可能**"，不是真预言——这条红线没变。
- **花钱**：每加一条路、每切一个走向、每轮对话都会调 DeepSeek（你的 key）。已做的省钱：结果存本地不重复生成、变体按需才生成、我自己测试时尽量走本地。**建议白天看一眼用量**。
- **递归地图**：分支很多/很深时，车道排布可能拥挤、标签可能叠（没做大规模压力测试）。这是下一步要打磨的。
- 移动端只是基本响应式；账号/云端按你的决定**没做**（不在本 PRD）。
- 这次用了子代理分块：地图（ui-pro-max）、未来自我对话（通用代理）由它们写，我做集成、接线和验证。
