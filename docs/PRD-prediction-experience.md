# PRD · 人生预测体验（v2 重做）

> 状态：草案，待 Jerry 评审
> 日期：2026-06-15
> 背景：现有预测被判定"太假、不细致"。本文把"什么样的预测才算真、才算细"定死，作为重做依据。
> 范围：只管"点开一条岔路后看到的那段人生"的生成与呈现。树/动画/引导框架不在本文重写范围，但会按需扩字段。

## 0. 定位红线（不可逾越）
- 这是"**贴着真实的你、可信的一种可能**"，不是真预言。任何 AI（含 MiroFish）都做不到对具体真人做准确预测——MiroFish 官方也承认其本质是"模式复制、无真实预知"。
- 因此**成功标准不是"算得准"，而是"足够真实可信、足够细致、不和你已知事实矛盾、尊重现实规律"**，让你看了觉得"这确实可能是我"。
- 凡是"保证/一定/注定"这类预言腔，禁止。

## 1. 现状问题（要解决的）
Jerry 的判定（四条全中）：
1. **缺现实锚点、太泛**（"假"的主因）：写"进了大厂"这种空话，没有真实公司/签证时间线/薪资档/城市/行业规律。
2. **不够密、不够深**（"不细致"的主因）：只有 4–5 个里程碑、每段一两句。
3. **维度太单一**：几乎只讲事业，没有感情/家庭/健康/财务/居住/身份。
4. **只有单一剧本、不能追问**：一条写死的路，点开节点不能问"然后呢/为什么"。

## 2. 目标 / 非目标
**目标**：让一条预测达到"现实锚点扎实 + 时间线密而深 + 多维度并行 + 多走向 + 可下钻"。
**非目标**：真预言；覆盖全球所有国家/行业的精确数据库（先覆盖用户填的国家/行业，靠模型知识 + 关键事实校准）；重做树动画与引导整体框架。

---

## 3. 需求详述

### R1 现实锚点（治"假"）
每条预测必须落到**具体、真实、可核对的现实**，禁止空话。具体要求：
- **签证/身份线**：若用户在国外且有签证身份，必须按真实规则推演。例：美国 H1B → 6 年上限、抽签风险、雇主 PERM → I-140 → 排期 → 绿卡（按国籍排期差异，如印度/中国 EB2/EB3 多年等待）→ 入籍。时间线要符合真实流程与大致年限。
- **薪资/财务线**：给该角色 + 该城市的**真实薪资区间与涨幅节奏**（如美国大厂 SWE 的 L3/L4/L5 总包档位、国内一线 P 序列），不是"年薪翻倍"这种空话。
- **职业阶梯**：用该行业**真实的层级与晋升节奏**（科技 IC：初级→高级→Staff；金融：Analyst→Associate→VP；学术：博后→AP→tenure 等）。
- **地点/行业规律**：具体城市（湾区/西雅图/纽约/上海…）、真实的行业周期与现实摩擦（裁员潮、签证被卡、行业下行）。
- **禁止泛词清单**：进了大厂、走上人生巅峰、实现逆袭、财务自由（除非给出具体数字/路径）。

> 实现提示：主要靠提示词要求"用具体真实锚点"，并对**高频强约束领域（美国工作签 → 绿卡时间线）内置一小段事实校准**喂给模型，避免它把签证写错。其余靠模型自身知识。

### R2 更密、更深（治"不细致"）
- **节点数**：单条预测 **6–10 个**关键时刻（现为 3–5）。跨度内可疏密不均：临近现在更密（每 1–2 年），远期可拉长。
- **每段深度**：每个节点 **3–5 句**，必须含至少：一个具体的人或机构、一个具体数字（薪资/年限/金额/比例）、一处内心或细节。
- **承接**：节点之间要有因果承接（这一步导致下一步），不是孤立片段。

### R3 多维度并行（治"单一"）
一条人生同时推进多条线，每个节点标注它主要触及哪些维度：
- 维度集合：`career 事业` `finance 财务` `relationships 感情/家庭` `health 健康` `housing 居住` `identity 身份/签证`。
- 要求：整条预测里 **career 之外至少再覆盖 3 个维度**，且 identity（对有签证的人）和 finance 必出现。
- 呈现：节点带维度标签；详情页可按维度看到各自的走势（沿用本地已有的领域指标曲线，但要和故事对得上）。

### R4 多走向 + 可下钻（治"单一剧本"）
- **多走向**：同一个选择给 **3 个版本**——`乐观 / 最可能 / 保守`。三者共享同一现实约束，但际遇与结局不同（如 H1B 抽中 vs 没抽中）。默认展示"最可能"，可切换。
- **下钻**：点开任意节点，可问 **"然后呢？" / "为什么会这样？" / "如果当时…"**，AI 基于该节点上下文继续展开更细的子情节或解释。每次下钻是一次新的生成调用。

## 4. 数据模型变化
- `Profile`：在现有基础上，明确这些字段进入预测（已具备 location/status）；**新增**可选：`industry 行业`、`targetGoal 你最想要的`（让走向有方向感）。
- `PathNode`：新增 `dimensions: LifeArea[]`（该节点触及的维度）。`story` 变长。
- `LifePath`：新增 `scenario: 'optimistic' | 'likely' | 'conservative'`；同一选择可挂三条 scenario 变体（或一条主 + 两条备选，按性能取舍，见 §6）。
- 下钻：新增 `NodeExpansion`（针对某节点的追问与展开），可缓存在该节点下。

## 5. UX 变化
- 详情页：时间线更长更密；每节点显示维度标签 + 更长故事；顶部可切 `乐观/最可能/保守`。
- 节点可点："然后呢 / 为什么 / 如果…" 三个快捷追问 + 自由输入，展开为子卡片。
- 维度：详情页保留并强化 5–6 条领域曲线，且与故事一致。
- 加载：因为更重，明确显示"AI 正在推演…"，并允许"先看最可能、其余按需生成"。

## 6. 生成方案与性能取舍（诚实）
- 更密 + 更深 + 多维 + 多走向 = **更长、更慢、更贵**。单条"最可能"预测预计 1.5–3k tokens、十几秒；三走向 ×3。
- 取舍策略（建议）：
  - 默认只生成 **"最可能" 一条**（6–10 节点、多维、带锚点）；`乐观/保守` 和 **下钻** 都 **按需点击再生成**，不预生成。
  - 模型：`deepseek-chat` 起步（快、便宜）；质量不够时该条切 `deepseek-reasoner`。
  - 现实锚点：注入一小段"签证时间线/薪资档"事实校准，降低硬伤。
- 仍保留：本地引擎先出结构和曲线、AI 文字异步补；失败回退本地。

## 7. 验收标准（拿 Jerry 的画像做测试用例）
画像：26 岁、已读完研、美国纽约、H1B、软件工程师、月薪 2–5 万、恋爱中。选择："跳槽去一线大厂"。
合格的"最可能"预测必须满足：
- [ ] 全程年龄 > 26，不重写过去，不与"已读完研/H1B"矛盾。
- [ ] **现实锚点**：出现真实签证里程碑（H1B 6 年/抽签/PERM/I-140/绿卡排期至少 2 项）、具体薪资档（如 L4/L5 总包数字）、真实层级节奏、具体城市。
- [ ] **密度/深度**：≥ 6 个节点；每节点 3–5 句、含具体人/数字/细节。
- [ ] **多维度**：career 之外覆盖 ≥ 3 维，且必含 identity（签证→绿卡）与 finance。
- [ ] **可信**：无"逆袭/巅峰"空话；有现实摩擦（如抽签未中、晋升卡壳）。
- [ ] 多走向（乐观/最可能/保守）可切；至少一个节点可"然后呢"下钻出更细内容。

## 8. 分期实现

## 10. 可对话 Agent（R5，新增 · Jerry 拍板"两个都要"）

调研后定的方向：对话 agent 不做成通用 chatbox，分两块，**先做杀手锏 R5a，再做辅助 R5b**。

### R5a 跟"未来的你"对话（杀手锏）
- **形态**：点开树上某条曲线（一个"未来的你"）→ 进入对话，与"走了这条路、X 岁的你"**第一人称**聊天。
- **扎根**：system prompt 注入该路径的全部上下文——`profile`（现状/身份/签证/所在地）+ 这条路已生成的 `summary`/`nodes`（关键时刻）+ 现实约束。AI 以那个未来自己的口吻回答（"当年我抽 H1B 没中，后来…"），不得与已生成的人生和已知事实矛盾。
- **可追问**：内置快捷问题"你后悔吗 / 当时怎么熬过来的 / 给现在的我一句话 / 如果当时选了别的"，也可自由输入。**这同时实现 R4 的"节点下钻/追问"**——两者合一。
- **底线**：仍是"一种可能的你"，不是真预言。价值参考 MIT Future You：重点是"与未来建立连接、帮你做更好的当下决定"，而非算准。

### R5b 规划助手（辅助）
- **形态**：常驻侧边栏/浮窗的 AI 助手，帮你**改树、加岔路、给建议**（"你还该考虑哪些选择""这条路要注意什么"），可触发现有生成与下钻。
- 偏工具属性，第二优先级。

### 实现要点
- 流式对话（DeepSeek `/chat/completions` stream）；每条消息把路径上下文作为 system 注入；控成本/限流，沿用 §6 取舍（按需、可回退）。
- 复用既有 `src/lib/enrich` 的提供商抽象与服务端密钥隔离；新增 `/api/chat` 路由 + 一个对话 UI。

### 分期更新（接 §8）
- **P3 = R5a「跟未来的你对话」**（吃掉原 R4 下钻）。
- **P4 = R5b 规划助手**。
