# i18n / Chinese strings

- The app is **bilingual (zh default + en)**. Every user-facing string goes through `t(...)`.
- **New UI strings**: add an English entry in `src/i18n/messages.ts`. **Additions-only** — never bulk-rewrite the dictionary.
- **Chinese-punctuation keys** (containing `，：（）「」…`) MUST be quoted as JS object keys.
- **Never** put ASCII double-quotes `"` inside Chinese text, and **never** use smart/curly quotes as JS string delimiters (only as content). A botched global replace once broke the build — use targeted edits.
- Interpolation: keys use `{name}` placeholders, passed via `t("…{n}…", { n })`.
- If a key already exists with the right meaning, reuse it (avoid duplicate keys — tsc/lint will catch dups).
