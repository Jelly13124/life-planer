# 预测可信度 v2 — 国籍 + 领域/国家自适应锚点 (2026-06-23, Route A 焦点①)

分支 `feat/goal-planning-mainline`。Route A 的生死线是"预测不假"。现 prompt(src/lib/enrich.ts)已很硬(禁用词/反预言/签证校准/具体锚点/摩擦/因果),但有两处系统性偏差,对**非美国科技路径**(含主用户:在中国的创业者)会变假或泛:

## 诊断(具体)
1. **输入是中国、锚点是美国科技。** 薪资输入是人民币档(5千/1万…),但 prompt 的"至少 2 处收入数字+职级"硬例是 `L5 $360k`/SWE→Staff/H1B/PERM。非美科技路径下,这把模型逼向"编个科技数字"或"含糊带过"。
2. **缺国籍/出生国。** VISA_US_FACTS 明说排期按出生国分、"不知道就把依赖写进故事"——我们根本没收集这个 prompt 点名要的输入。

## 改动
### 1) Profile + 录入:加 nationality(出生国/地区)
- types.ts `Profile.nationality?: string`(自由文本/轻选择,如"中国大陆/印度/美国…")。可选,不破坏旧档(normalize 不需迁移,可选字段)。
- profile.ts:`buildSnapshot` 与一个新的"背景事实"行带上国籍;deriveAreas 不变。
- Onboarding 录入加一项"国籍/出生国"(选填,给常见项 + 自由填)。i18n。

### 2) enrich.ts prompt:领域/国家自适应锚点(核心)
- facts 里注入国籍:`国籍/出生国：X`(让排期等逻辑可落到具体)。
- **货币**:把"在美国用美元"改成"用主角所在国/文化的货币与口径"(中国用人民币、美国用美元、欧洲欧元…),薪资写当地真实档位。
- **职级/里程碑锚点泛化**:不再把 SWE/H1B/$360k 当作 THE 例子。改成:"用 {occupation} 在 {所在国} 的真实职业阶梯、真实机构、真实里程碑与货币"。给**多国/多领域**的简短示例(科技大厂职级、体制内/公务员路径、创业里程碑[融资轮次/营收/团队规模]、教师/医生/学术、金融),让模型按主角实际领域选,而不是默认美国科技。
- **美国签证块保持按 isUSVisa 门控**(已是);**非美路径不强加美国里程碑**——明确"若不涉及移民,身份维度可不出现,用所在国的现实约束(户口/编制/行业周期/房价)替代"。
- "现实锚点(整条至少满足)"的 (a) 身份里程碑改为"仅当涉及移民/签证时";(b)(c)(d) 收入数字/机构/城市保留但要求用**主角所在国**的真实口径。

### 3) chat.ts(和未来的自己对话)同步喂国籍(口径一致,次要)。

## 不动
- 不碰禁用词表/反预言/摩擦/因果/多维度这些已生效的硬约束。
- 不动数字曲线引擎、forkDelay 逻辑、schema。
- 纯服务端 prompt + 一个可选 Profile 字段 + 录入 UI;确定性不受影响。

## 验收(eval 驱动,key 已配)
用 prediction-quality-reviewer / predict-eval 在多 persona 上跑前后对比:
- (a) 在中国的创业者(主用户):应出现人民币、融资轮次/营收/团队规模等真实里程碑,无 H1B/$ 乱入。
- (b) 美国 OPT 上的 CS 硕士:US-visa 块照常、质量不回退。
- (c) 三线城市教师 / (d) 金融 analyst:用各自领域真实阶梯。
判据:非美 persona 的"现实锚点具体度"提升、不再出现错配的美国科技锚点;美国 persona 不回退。改 prompt 有回归风险 → 必须 eval 确认。

## 风险
prompt 是已调好的;泛化锚点有"反而更泛"的风险。靠多 persona eval + 保留所有现有硬约束 + 只把"美国专属例子"替换成"按主角领域/国家"来控。
