# 职业决策风格测试 v2：可信测量外壳 + Web/App 传播闭环

**日期：** 2026-07-10
**状态：** 用户已逐段确认，待书面复核
**范围：** Web 28 题完整版、App 12 题快测、连续四轴评分、16 类标签、分享/朋友对比、第一方匿名漏斗
**替代：** 现有 `lifePathCode` 职场人格测试及其固定可行度语义

## 1. 背景与结论

现有测试具有可传播的身份标签结构，但不具备将自己称为可靠人格测评的证据：

- 四轴未经真实样本验证。
- 每轴 7 道同意度题以 4:3 混合方向计分，仍受一律同意/否定影响。
- 连续分数被直接二分，平分时强制落入 `SDLG`。
- 每类携带写死的“现实可行度”，却显示为“AI 粗估”。
- `tempo` 字母被直接映射成金融风险偏好。
- 部分标签写入题目没有测量的“大厂、体制内、套利”等环境信息。
- Web 只有 SVG 下载，没有完整的一键分享、动态 OG、朋友对比和可测量漏斗；App 没有测试。

产品决策是保留 MBTI 式的传播结构，但停止借用心理测评权威。统一名称改为 **“职业决策风格测试”**，结果表达为当前倾向而非固定人格。

当前没有真实用户，因此不做旧结果兼容、不保留旧数据解释、不提供静默重算迁移。旧模块、旧字段和旧结果入口在新版本完成后直接删除。

## 2. 产品承诺与非目标

### 2.1 产品承诺

测试帮助用户观察自己在职业选择中的四组决策倾向，并生成便于认领、分享和比较的四字母代码与中文标签。它是自我探索工具，不是心理诊断、能力评级或未来预测。

### 2.2 非目标

- 不宣称心理学诊断效力或科学常模。
- 不输出人格等级、成功率、路线可行度或未来概率。
- 不用测试结果自动决定金融风险偏好、人生路线、事件或结局。
- 不生成关系匹配度、适不适合、谁更好等判断。
- 首版不做自适应测试；有真实样本后再评估。
- 不接入第三方分析平台。

## 3. 四个连续维度

沿用四字母体系，重新限定解释边界：

| 轴 | A 极 | B 极 | 测量范围 |
|---|---|---|---|
| 行动 `tempo` | `F` 先试再调 | `S` 先验证再动 | 信息不完整时的职业行动方式 |
| 路径 `focus` | `D` 集中深耕 | `W` 多线探索 | 时间与机会在单线/多线之间的配置 |
| 借力 `engine` | `B` 自主掌控 | `L` 平台借势 | 更偏自主定义规则还是利用组织资源 |
| 取向 `drive` | `G` 保障回报 | `V` 意义实现 | 做职业取舍时优先保护的价值 |

这些维度是连续倾向，不假设两端互斥，也不推断用户在所有情境下都稳定如此。

## 4. 题目设计

### 4.1 题目形态

旧版“非常符合—完全不符合”改成双端行为选择。每题包含：

- 一个具体职业情境，优先描述过去一年或最近几次真实行为。
- 左右两个都合理、社会吸引力接近的行为端点。
- 五档选择：明显偏左、稍偏左、视情况而定、稍偏右、明显偏右。
- `leftPole`、`rightPole` 明确映射到同一轴的两端。

题目不使用“天生、注定、一定、最怕、底线、十年磨一剑”等极端身份措辞，不把理想自我当成实际行为。

### 4.2 题集

- Web 完整版：28 题，每轴 7 题。
- App 快测：从完整题库标记每轴 3 道覆盖面最广的代表题，共 12 题。
- 每轴配置 1 道额外二选一平分追问题，不计入 12/28 的基础题数。
- 快测和完整版共享题目 ID、轴定义与评分函数。
- App 结果标记 `quick`，Web 完整结果标记 `full`；完整结果可以覆盖快测结果。

## 5. 评分与边界

### 5.1 规范化

每题 UI 值为 `-2..2`。评分时按 `leftPole/rightPole` 转换到该轴的统一方向，再求和。每轴标准化成 `0..100`：

- `100` 表示明显偏 A 极。
- `0` 表示明显偏 B 极。
- `50` 表示基础题恰好平衡。

不把此分数称为概率、准确率或置信度。

### 5.2 字母与倾向强度

- `score > 50` 取 A 极字母。
- `score < 50` 取 B 极字母。
- `score === 50` 时只对该轴显示平分追问题；用户明确选择后决定字母，展示分数仍为 `50`。
- `45..55`（含边界）显示“轻微倾向”。
- 其余显示“明显倾向”。
- 不存在默认字母，不存在 `TIE_DEFAULT`。

### 5.3 结果依据

本地结果页从非中立答案中选出绝对贡献最大的三题，尽量覆盖不同轴，用来解释“哪些选择最影响结果”。原始答案不进入公开链接、分析事件或 OG 图片。

## 6. 16 类传播标签

| 代码 | 标签 |
|---|---|
| FDBG | 务实攻坚者 |
| FDBV | 信念开拓者 |
| FDLG | 借势攀登者 |
| FDLV | 组织破局者 |
| FWBG | 多线经营者 |
| FWBV | 自由开拓者 |
| FWLG | 机会整合者 |
| FWLV | 跨界理想家 |
| SDBG | 稳健匠人 |
| SDBV | 长期创造者 |
| SDLG | 深耕积累者 |
| SDLV | 价值深耕者 |
| SWBG | 稳健多面手 |
| SWBV | 自在探索者 |
| SWLG | 稳健多栖者 |
| SWLV | 从容连接者 |

每类内容只包含：

1. **优势：** 这种决策方式在哪些情况下有效。
2. **代价：** 过度使用时容易付出的现实成本。
3. **决策建议：** 面对重大选择时适合补足的盲区。
4. **张力：** 四个维度之间最典型的内在拉扯。

全部标签保持相近的社会吸引力。内容不得加入题目没有测量的行业、单位、身份、城市、收入或未来事件。

## 7. 共享领域模型

旧 `packages/core/src/lifePathCode/` 替换成 `packages/core/src/decisionStyle/`：

- `axes.ts`：轴、字母、标签与分数解释。
- `questions.ts`：28 道基础题、12 题 quick 标记、4 道平分追问题。
- `scoring.ts`：规范化、平分检测、代码、倾向强度和结果依据。
- `types.ts`：16 类标签与结果文案。
- `shareToken.ts`：公开结果的最小载荷编码/解码与校验。
- `index.ts`：Web/App 共用出口。

核心结构拆成可同步摘要与仅设备本地详情：

```ts
type DecisionStyleSource = "quick" | "full";

interface DecisionStyleAxisScores {
  tempo: number;
  focus: number;
  engine: number;
  drive: number;
}

interface DecisionStyleSummary {
  version: 2;
  source: DecisionStyleSource;
  code: string;
  scores: DecisionStyleAxisScores;
  completedAt: string;
}

interface DecisionStyleLocalDetail {
  version: 2;
  answers: { questionId: string; value: -2 | -1 | 0 | 1 | 2 }[];
  tieBreaks: Partial<Record<keyof DecisionStyleAxisScores, "a" | "b">>;
}
```

`Profile` 删除 `lifePathCode` 与 `lifePathAnswers`，新增可选 `decisionStyle?: DecisionStyleSummary`。它可随现有云同步跨设备，但不含原始答案。

`DecisionStyleLocalDetail` 使用独立存储键：Web 存 `localStorage/sessionStorage`，App 存 AsyncStorage。它不进入 `LifeTree`、Supabase 云同步、AI 请求、分享 token 或分析事件。完成测试后只保留本地结果解释所需的数据；用户重置数据时一起删除。

结果优先级：

- `full` 永远可以覆盖 `quick`。
- 已有 `full` 时，新的 `quick` 只显示本次预览，不降级覆盖完整结果。
- 同来源重测以显式完成的最新 `completedAt` 覆盖。
- `completedAt` 不进入公开 token。

## 8. AI 数据流

`DecisionStyleSummary` 不再改写 `Profile.riskAppetite`。用户仍在资料引导中独立选择金融/现实风险偏好。

当且仅当 `decisionStyle.version === 2` 时，AI 提示词加入一行低权重背景：

- 只描述四轴分数对应的倾向。
- 明确它是自报的职业决策风格，不是事实或人格定论。
- 明确不得据此创造地点、职业、身份、财务、关系、疾病或未来事件。
- 不注入中文类型昵称，避免模型按标签刻板扩写。
- AI 请求序列化必须显式构造最小摘要，不得把本地原始答案或平分选择传给后端。

用户重测只影响之后的新 AI 请求，不自动重写已生成的人生树。

## 9. Web 体验

### 9.1 测试

`/test` 增加短介绍：用途、预计时长、本地计算、非心理诊断。流程为基础题 → 必要平分追问 → 结果。

中途答案保存在 `sessionStorage`，刷新或返回后可继续；完成后清除进度并保存最终结果，供 onboarding 消费。

### 9.2 结果

结果卡展示：

- 四字母代码与中文标签。
- 四轴条形分数及“轻微/明显倾向”。
- 优势、代价、决策建议、张力。
- 本地三条结果依据。
- “当前倾向，不是固定人格”的说明。

操作顺序：一键分享 → 复制链接 → 保存 PNG → 继续生成人生树 → 重新测试。

### 9.3 路由与分享

- `/style/[code]/[token]`：个性化公开结果。
- `/compare/[left]/[right]`：朋友四轴对比。
- 对旧 `/t/[code]` 不保留产品兼容；上线时删除旧路由。

公开 token 只含 `version`、`source`、`code` 和四个取整分数。分享发生时必须联网，客户端把最小载荷发给服务端签名 API；服务端重新计算代码、验证 `0..100` 范围并返回带 HMAC 的 token。签名密钥只在服务端环境变量中。

动态 OG 与 PNG 使用相同的服务端结果卡渲染。签名无效、版本未知、代码与分数不一致时不展示结果，返回测试入口。

朋友打开分享页后点击“测完和 TA 比”，邀请 token 只在本次测试流程中传递。完成后进入 compare 页，展示接近轴、最大差异轴和两种决策方式，不输出匹配分。

## 10. App 轻量体验

### 10.1 首次引导

App onboarding 首屏提供：

- 主操作“开始 12 题快测”。
- 次操作“先跳过”。
- 本地计算、可稍后补测的说明。

完成后显示精简结果，再进入现有个人资料引导。跳过时 `decisionStyle` 保持缺省，不影响后续功能。

### 10.2 “我”页面

- 有结果：显示精简风格卡、来源（快测/完整）、重测、分享。
- 无结果：显示补测入口。
- 重测完成后覆盖旧结果，不保留历史版本。

### 10.3 离线与分享

测试和本地保存离线可用。公开分享需要联网：

1. 调用 Web 后端取得签名 token。
2. 生成公开链接及统一 PNG URL。
3. 下载 PNG 到临时缓存并调用系统分享。
4. 图片分享失败时自动退回链接分享。
5. 分享结束后清理临时文件，不永久写入相册。

实现时使用 Expo SDK 56 官方兼容的文件与分享模块版本，并在真实 iPhone/TestFlight 验证。

## 11. 第一方匿名漏斗

新增 `/api/style-events` 与 Supabase `style_events` 表。允许事件：

- `style_view`
- `style_start`
- `style_skip`
- `style_complete`
- `style_share`
- `style_share_open`
- `style_compare_start`
- `style_compare_complete`
- `style_continue_tree`

事件只保存：

- `event`
- `surface`：`web | app`
- `source`：`direct | shared | compare`
- `test_version`
- 服务端 `created_at`

不保存 flow ID、IP、账号、设备 ID、姓名、答案、分数、代码、token 或自由文本。漏斗使用时间区间内的聚合计数，不做用户级追踪。

API 使用严格枚举、请求大小限制和限流。数据库写入失败返回成功型 no-op，不阻塞产品流程。Supabase 定时任务每天删除 30 天前事件。隐私政策披露匿名聚合事件、字段与保留期限。

## 12. 隐私与可访问性

- 测试本身不需要网络，原始答案只存在设备本地独立存储中；可选云同步只同步四轴摘要。
- 只有主动分享时，类型与四轴取整分数进入公开链接。
- 进入人生树前明确说明决策风格摘要会随资料用于 AI 生成。
- Web 与 App 均提供可见的隐私政策入口。
- 所有问题、选项、进度、结果条、分享和重测控件提供语义角色、可访问名称和状态。
- 支持键盘、VoiceOver、动态字体、减少动态效果和至少 44pt 的移动触控区域。
- 颜色不是表达轴方向和强弱的唯一方式。

## 13. 异常与降级

- 测试中断：恢复本地进度。
- 精确平分：只追加对应轴追问题，不默认归类。
- 无效分享 token：不渲染伪造结果，提供重新测试入口。
- OG/PNG 失败：仍可分享文字链接。
- App 离线：可测试和保存；分享提示联网后重试。
- 统计失败：静默 no-op。
- 用户跳过：AI、树和其他 onboarding 正常工作。
- 未配置分享签名密钥：公开分享入口显示暂不可用，不生成不受验证的 URL。

## 14. 测试策略

遵循 TDD，先写失败测试再改实现。

### 14.1 纯领域测试

- 28 题每轴 7 题；quick 每轴 3 题。
- 每题左右端点属于同一轴且互为两极。
- 分数始终在 `0..100`。
- 全中立返回四个待追问轴，不返回默认代码。
- 只有平分轴需要追问；追问后得到完整四字母代码。
- `45..55` 正确标记轻微倾向。
- 16 个代码全覆盖且不存在 `feasibility`。
- 结果依据只来自本地答案。
- token 最小载荷、签名、往返和篡改拒绝。
- AI 上下文不包含类型昵称、不产生风险偏好映射。
- AI 客户端请求体不包含原始答案、平分选择或本地结果依据。

### 14.2 Web 测试

- 介绍 → 28 题 → 平分追问 → 结果。
- 刷新恢复进度，完成后清理草稿。
- 分享、复制、PNG、动态 OG。
- 分享接收者 → 测试 → 双结果 compare。
- compare 不输出匹配分和优劣判断。
- onboarding 正确接收完整结果。
- 事件请求不含答案、分数、代码、token 或资料。

### 14.3 App 测试

- “我”页面补测、重测、来源标识。

## 15. 产品验证计划

首轮朋友测试记录：测试开始、完成、分享、分享打开、对比完成和进入人生树的聚合数量，同时做录屏与访谈。

测评质量升级需要后续真实样本：

1. 10–15 位目标用户逐题认知访谈。
2. 300–500 人首轮样本检查题目表现、内部结构和群体偏差。
3. 至少 100 人间隔约两周重测。
4. 在证据形成前始终使用“风格测试/当前倾向”，不升级为科学人格测评承诺。
