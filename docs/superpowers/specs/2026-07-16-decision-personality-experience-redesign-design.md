# 决策人格测试与分享体验重设计

日期：2026-07-16
状态：已确认设计，待实施计划

## 1. 背景

现有职业决策风格测试已经具备题库、计分、结果同步、签名分享和朋友对比闭环，但呈现仍像内部数据工具：答题页是五个长文本按钮，结果页突出原始代码与轴分数，分享卡由深色渐变和四条进度条组成。用户完成后很难产生“这说的就是我”的认同，也缺少主动转发给朋友的理由。

本次改造采用“类似 MBTI 的娱乐型人格体验”，但不复制现有 MBTI 产品的角色、命名或视觉。产品对外强调有记忆点的四字母决策人格、半夸半损的结果文案和统一角色形象；底层仍明确它只是当前职业决策倾向，不是固定人格、心理诊断或科学人格测评。

## 2. 已确认的产品方向

- 整体气质：娱乐型人格测试。
- 文案尺度：半夸半损，好笑、有共鸣，但不羞辱用户。
- 身份主视觉：四字母代码继续作为最大标题，中文人格名作为第二层。
- 答题方式：左右真实选择 + 中间五档刻度，点击后自动进入下一题。
- 分享钩子：“我是 FDBG，你是什么？”并引导朋友测试和对比。
- 角色视觉：16 个现代编辑插画小人，成熟、有个性、不幼态。
- 首轮范围：Web 完整测试、App 12 题快测、结果页、PNG/OG 分享卡和朋友对比页保持统一。

## 3. 目标与非目标

### 3.1 目标

1. 让答题过程从“填写表单”变成有节奏的快速人格揭晓。
2. 让用户在结果首屏先看到身份、角色和一句能转述的梗，而不是内部轴名和数字。
3. 让分享卡脱离数据报告感，成为愿意保存和发给朋友的社交人格海报。
4. 让 Web、App、公开页面和生成图片共用同一套人格内容，不再跨端漂移。
5. 保持现有计分、隐私、签名 token、云同步和朋友对比兼容。

### 3.2 非目标

- 不修改 28 道完整题、12 道快测题或四轴计分算法。
- 不把结果包装成科学验证的人格诊断。
- 不新增账号要求、数据库表或用户级追踪。
- 不为每个平台维护独立人格文案。
- 不照搬 16Personalities 或其他人格产品的角色、色板和页面构图。

## 4. 端到端体验

### 4.1 开场

主标题使用“测测你的决策人格”，说明为“12 道选择题，看看你做重大决定时像哪种人”。背景可露出数个人格角色的局部剪影，建立完成后的揭晓期待。

“本地计算”“不是心理诊断”“可稍后重测”等信息保留，但降为页面底部说明，不与主行动争夺注意力。

### 4.2 答题

每屏只显示一题：

1. 顶部为细进度条、返回按钮和 `03 / 12` 或 `08 / 28`。
2. 中部显示问题，以及左、右两个真实决策选项。
3. 两端选项之间显示五个可触控圆点，对应 `-2 / -1 / 0 / 1 / 2`。
4. 页面不重复显示“明显更像左边”“比较符合右边”等布局术语。
5. 用户点击圆点后，圆点放大并短暂高亮；约 200ms 后自动进入下一题。
6. 用户可以返回上一题并修改答案。修改后仍按原规则保存本地进度。
7. VoiceOver 为五档提供完整可访问名称，例如“强烈偏向：先试一次再调整”，视觉上无需展示这些长标签。

最后一题选择后直接进入结算，不再要求额外点击“完成”。精确平分时出现“加赛题”，沿用左右二选一的视觉，不使用“平分追问”“轴”等内部术语。

### 4.3 揭晓

完成后先显示短暂角色剪影，再进入结果。减少动态效果开启时跳过揭晓动画，直接显示完整结果。

结果首屏按以下优先级展示：

1. 超大四字母代码。
2. 中文人格名。
3. 对应人格角色。
4. 一句半夸半损的核心梗句。
5. 一个高光、一个翻车点和一个可执行建议。
6. 主操作“分享我的人格”，次操作“看看我为什么是这个类型”。

四轴分数、每轴两端含义、结果依据和“当前倾向，不是固定人格”放在可展开详情中。内部字段名 `tempo`、`focus`、`engine`、`drive` 不直接面向用户。

FDBG 示例：

- 代码：`FDBG`
- 名称：务实攻坚者
- 梗句：你不是没耐心，只是觉得今天能解决的事，不该开三次会。
- 高光：目标一清楚，你通常是最先把事情推起来的人。
- 翻车：推进太快时，别人和后手可能还没跟上。
- 建议：重要决定前，强制找一个人唱反调。

## 5. 分享与对比

### 5.1 竖版分享海报

App 系统分享和 Web“保存 PNG”使用 4:5 竖版海报，目标尺寸为 1080×1350：

- 顶部：LifePlanner 小标识和“决策人格”。
- 主体：人格角色、超大四字母代码和中文人格名。
- 内容：核心梗句、一个高光和一个翻车点。
- 底部：“我是 FDBG，你是什么？”、测试入口和编码本次签名结果页的二维码。
- 最底部小字：“当前自报倾向，不是固定人格或心理诊断。”

隐私说明不再占据海报正文。公开页面仍可在正文中完整说明分享内容不含原始答案。

### 5.2 OG 图片

社交链接预览继续使用 1200×630 横版，不把竖版海报强行裁切。横版与竖版共用同一人格展示数据和角色资产，只采用不同构图。

### 5.3 朋友对比

朋友完成测试后，对比页并排展示两个角色和代码。对比文案优先使用可转述的关系句，例如“你负责踩油门，TA 负责确认前面有没有路”，再展开最接近维度和最大差异维度。继续不输出匹配率、优劣或关系预测。

## 6. 16 型角色系统

角色采用统一的现代编辑插画语言：稳定线宽、暖白基底、低饱和配色、轻微纸张颗粒和清晰轮廓。角色可以呈现不同外貌，但类型不绑定性别、年龄、职业、收入或文化身份。

四个轴控制视觉语法：

- `F / S`（先试再调 / 先验证再动）：向前迈步、快速出手 / 稳定站姿、先观察。
- `D / W`（集中深耕 / 多线探索）：聚焦一个核心道具 / 多个方向与线索环绕。
- `B / L`（自主掌控 / 平台借势）：亲手握住工具和控制权 / 借助阶梯、结构或协作网络。
- `G / V`（保障回报 / 意义实现）：锚点、积木、储备等保障意象 / 火花、旗帜、光源等意义意象。

这些规则用于保持系统一致，不代表简单拼装换色。每个代码仍需独立构图、姿态和标志性道具，并在分享卡小尺寸下保持可识别轮廓。

首轮交付 16 张统一画布、透明背景的高分辨率角色资产。Web、App、PNG、OG 和朋友对比页共用同一资产编号；缺少或加载失败时使用统一品牌剪影，文字结果始终完整可用。

## 7. 内容与组件边界

共享领域层维护每种代码的展示资料：

```ts
interface DecisionPersonalityPresentation {
  code: DecisionStyleCode;
  label: string;
  tagline: string;
  highlight: string;
  roast: string;
  advice: string;
  characterId: string;
}
```

现有 `DecisionStyleSummary`、题目、原始答案和计分函数保持不变。展示资料是确定性的静态内容，不依赖 AI 临时生成，保证同一代码在所有平台、所有分享中使用同一身份表达。

组件职责拆分为：

- 双端五档答题器：只负责显示选项、提交数值和自动推进。
- 测试流程控制器：负责进度恢复、返回、加赛和完成。
- 人格主视觉：负责代码、名称、角色和核心梗句。
- 结果详情：负责轴解释、依据、免责声明和重测。
- 分享画布：读取同一展示资料，分别渲染 portrait 与 OG 变体。
- 对比摘要：把两个公开结果转成关系文案，不修改任何一方类型。

App 分享数据流保持为：本地结果 → 服务端验证并签名 → 公开 PNG URL → 下载临时文件 → 系统分享。系统接口支持附加文字时同时附上签名结果链接；图片失败时自动回退到签名链接。公开 token 仍只含版本、来源、代码和四轴整数分数。

## 8. 状态、异常与可访问性

- 本地草稿存在时恢复到第一道未完成题；已作答题可返回修改。
- 快速连续点击时只接受当前题的第一个有效选择，防止跳过题目。
- 自动推进期间提供短暂锁定和选中反馈，不显示全屏加载。
- 图片加载失败使用剪影；分享 PNG 失败使用链接；签名失败提示联网后重试。
- 所有刻度触控目标至少 44pt，选中状态同时使用尺寸、描边和可访问状态表达。
- 支持动态字体；长中文不截断人格名、梗句和行动建议。
- 支持系统减少动态效果，动画不承载必要信息。
- 不使用颜色作为轴方向、选择状态或类型差异的唯一表达。

## 9. 验证策略

### 9.1 自动化

- 五档 UI 值与 `-2..2` 的映射准确。
- 选择后只自动前进一题，返回修改不会新增重复答案。
- 最后一题自动结算；平分时只进入需要的加赛题。
- 中断恢复、完成后清理草稿和 quick/full 覆盖规则保持不变。
- 16 个代码均有完整展示资料和合法角色编号。
- Web、App 适配层与分享画布读取同一人格展示资料。
- portrait PNG 为 1080×1350，OG 为 1200×630，签名无效仍返回 404。
- 图片下载失败继续回退签名链接。

### 9.2 视觉与设备验证

- Web 对答题、结果、公开页、对比页、portrait 和 OG 做关键尺寸截图回归。
- 检查最短与最长中文人格名、梗句和建议在小屏及图片中的换行。
- App 完成 TypeScript、Expo 配置和打包验证后发布 iOS OTA。
- 在真实 iPhone/TestFlight 检查触控、自动推进、返回、动态字体、VoiceOver、减少动态效果、PNG 分享和链接回退。

## 10. 完成标准

- Web 与 App 不再显示五个重复长答案按钮，且答题可连续完成。
- 结果首屏不显示内部英文轴名，四字母、中文人格名、角色和梗句成为主视觉。
- 16 个类型均拥有独立、统一风格的角色和半夸半损文案。
- App、Web、公开页面、竖版 PNG 和横版 OG 对同一代码显示一致内容。
- 分享卡明确包含“我是 X，你是什么？”并能进入测试与朋友对比闭环。
- 现有计分、旧结果、云同步、签名校验、隐私边界和链接回退继续通过。
- 自动化验证通过，并完成至少一轮真实 iPhone 分享测试。
